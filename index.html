Using Dairy with Roadrunner 1.0

Dairy's service framework is tighly bound to the OpMode lifecycle and all services depend on Loop() occuring many times a second. By default, Roadrunner 1.0 runs code in a blocking loop, not letting the actual Loop() code of your OpMode run.

For users who are not using Mercurial as a commandbase, you will only need something to correct the blocking actions runner. This can be done with this sample Dairy service. It changes Actions.runBlocking() to Silkroad.runAsync()
